面向对象的特征

还记得第一次学面向对象编程的时候，自己就被这什么面向对象的三大特征给吓住了，这是啥呀，不会呀，这可咋整啊，现在当然是不会有这种感觉了，就再回顾一下吧。

  1. 封装

    • 套话是这么说的： 封装，也就是把客观事物封装成抽象的类，并且类可以把自己的数据和方法只让可信的类或者对象操作，对不可信的进行信息隐藏。
    • 我现在的想法是封装就是只暴露出有用的信息，让不同的封装对象独立起来，对使用者和安全都有好处，还可以把可以复用的代码或者方法封装成单独的方法或者类。

  2. 继承

    1. 可以使用现有类的所有功能，并在无需重新编写原来的类的情况下对这些功能进行扩展。
    2. 继承有点像字面意思，有的语言可以多继承(c++)，有的语言用实现的方式来取代多继承(java)，继承了父类，子类可以调用父类的public方法对象等。

  3. 多态

    1. 多态性（polymorphisn）是允许你将父对象设置成为和一个或更多的他的子对象相等的技术，赋值之后，父对象就可以根据当前赋值给它的子对象的特性以不同的方式运作
    2. 实现多态，有二种方式，覆盖，重载。
      • 覆盖，是指子类重新定义父类的虚函数的做法。
      • 重载，是指允许存在多个同名函数，而这些函数的参数表不同（或许参数个数不同，或许参数类型不同，或许两者都不同）。

final, finally, finalize 的区别

  1. finnal

    • final在Java中是一个保留的关键字，可以声明成员变量、方法、类以及本地变量。一旦你将引用声明作final，你将不能改变这个引用了，编译器会检查代码，如果你试图将变量再次初始化的话，编译器会报编译错误。
    • final变量是只读的。
    • 方法前面加上final关键字，代表这个方法不可以被子类的方法重写
    • final类通常功能是完整的，它们不能被继承。像是String类，可以保证线程安全。
    • finnal关键字的好处
      1. final关键字提高了性能。JVM和Java应用都会缓存final变量。
      2. final变量可以安全的在多线程环境下进行共享，而不需要额外的同步开销。
      3. 使用final关键字，JVM会对方法、变量及类进行优化。
    • 不可变类
      1. 它们的对象是只读的，可以在多线程环境下安全的共享
      2. 不用额外的同步开销

  2. finally

    • 配合try使用，大部分情况下，执行了try块的话(没执行肯定是没戏的)，finally都会执行(包括return)，当然不包括System.exit(0);，断电等情况。
    • 可以用作释放资源(java8有了try with resource)，或者用来保证某段代码必须执行等操作。
    • 至于其他骚操作的代码并不想做讨论。在写代码应该尽量避免这种歧义代码。

  3. finalize

    • java提供finalize()方法，垃圾回收器准备释放内存的时候，会先调用finalize()。
    • 可以用来做内存的释放(JNI?)，但是并不能保证这个方法在什么时候会运行，还是老老实实的用try…finally吧

int 和 Integer 有什么区别

int integer
基本数据类型 包装类
直接存储数据值 Integer实际是对象的引用
默认值是0 默认值是null

  1. 对于两个非new生成的Integer对象，进行比较时，如果两个变量的值在区间-128到127之间，则比较结果为true，如果两个变量的值不在此区间，则比较结果为false

    Integer i = 100;
Integer j = 100;
System.out.print(i == j); //true

Integer i = 128;
Integer j = 128;
System.out.print(i == j); //false

    原因是常量池，会从缓存中取值

重载(Overload)和重写(Override)的区别

  1. 重写

    class Animal{
    public void move(){
         System.out.println("动物可以移动");
    }
}

class Dog extends Animal{
    // 这个注解表示这个方法是重写的
    @Override
    public void move(){
        System.out.println("狗可以跑和走");
    }
}

  2. 重载

    public class Overloading {

    public int test(){
        System.out.println("test1");
        return 1;
    }

    public void test(int a){
        System.out.println("test2");
    }

    //以下两个参数类型顺序不同
    public String test(int a,String s){
        System.out.println("test3");
        return "returntest3";
    }

    public String test(String s,int a){
        System.out.println("test4");
        return "returntest4";
    }
}

抽象类和接口有什么区别

参数 抽象类 接口
方法实现 它可以有默认的方法实现 接口完全是抽象的。它根本不存在方法的实现
使用 使用extends来继承抽象类。如果子类不是抽象类的话，它需要提供抽象类中所有声明的方法的实现。 子类使用关键字implements来实现接口。它需要提供接口中所有声明的方法的实现
构造器 抽象类可以有构造器 接口不能有构造器
与正常Java类的区别 除了你不能实例化抽象类之外，它和普通Java类没有任何区别 接口是完全不同的类型
访问修饰符 抽象方法可以有public、protected和default这些修饰符 接口方法默认修饰符是public。你不可以使用其它修饰符。java8新增了default修饰符
main方法 抽象方法可以有main方法并且我们可以运行它 接口没有main方法，因此我们不能运行它。
多继承 抽象方法可以继承一个类和实现多个接口 接口只可以继承一个或多个其它接口
速度 它比接口速度要快 接口是稍微有点慢的，因为它需要时间去寻找在类中实现的方法。
添加新方法 如果你往抽象类中添加新的方法，你可以给它提供默认的实现。因此你不需要改变你现在的代码。 如果你往接口中添加方法，那么你必须改变实现该接口的类。

session 与 cookie 区别

  1. cookie存在于浏览器上，以key-value的形式存在，用于存储某些信息(Token，临时数据)，在客户端发起请求时(GET,POST等)会自动带上同域名同路径的cookie发往服务器。
  2. session存在与服务器上，依赖cookie生存，依赖唯一的cookie值(session_id)来确定用户的身份，从而进行后续的操作。

MVC 设计思想

  1. Model(模型),View(视图), Controller(控制器)
  2. 它将业务逻辑与展现分离开来，避免了将业务逻辑与展现混杂在一起带来的显示的不一致性和业务逻辑代码重复地分布在用于展现的代码中。
  3. 层次清晰，易于开发者对这三个部分分工与协作，易于维护者识别不同的层次实施不同的维护策略。
  4. 系统具有更好的重用性，包括用户界面的重用和业务逻辑处理包的重用，特别是业务逻辑处理包，如果遵循相应的java规范，它不仅可以在web应用程序中使用，而且可以在包括桌面、分布式环境下的得到重用。
  5. 系统更易于扩展和移植。
  6. 系统更易于维护。
  7. 采用MVC模式开发的系统更健壮。
